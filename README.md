#Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれなんでしょうか？  
### リモートリポジトリ
GitHubなどのインターネット上にある共有用のリポジトリ。ローカルリポジトリで行った変更をpushすることでリモートリポジトリに内容が反映させることができる。  

### ローカルリポジトリ
自分のパソコンの中にあるリポジトリ。実際にファイルを編集したり、変更内容を記録したりする作業を行う場所。  


## プッシュとマージの違いは何でしょうか？  
pushはローカルでの変更をリモートに共有する操作で、mergeはローカル内でブランチの変更を別のブランチに取り込む操作。
リモートだとプルリクエストをマージするという。   
↳コードの変更を他のメンバーに通知し、レビューを行うことで品質を保つための重要な機能

### push  
ローカルでの変更をリモートへアップロードして共有する  
↳ローカル→リモート  

### merge  
ブランチの変更を別のブランチに取り込む  
↳ローカル内  

## コミットとプッシュの違い  
commitはステージングした変更をローカルリポジトリに記録する操作で、pushはローカルでの変更をリモートに共有する操作。  



## コミットのメッセージはどのように書いてあげるのが最適でしょうか？  
変更した内容を一目でわかるような書き方にする  
〈例〉  
git commit -m 'ボタンの色を水色に変更'  

## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？  
ローカルでマージするフローは自分の㍶のローカルの中だけでブランチを反映させることで、プルリクエストでマージするフローは一度リモートにブランチをあげてほかのメンバーにレビューを行ってもらう事で安全にマージするやり方  



## コンフリクトを起こしてしまった場合、どう対処すべきですか？  
1.先にマージされた変更内容を取り込む  
2.後にマージしようとしている変更内容を取り込む  
3.どちらの変更内容も取り込む
  
状況によって上記の方法で変更を取り込む必要がある。
自分以外の人が書いたソースコードの内容を把握し、ソースコードを書いた人と相談しながら作業を進める。
